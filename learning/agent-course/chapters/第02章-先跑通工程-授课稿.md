# 第02章：先跑通工程，再谈高级能力（授课稿）

## 一、课程信息

- 建议时长：90 分钟
- 本章目标：
  - 学员能完成项目构建与测试
  - 学员掌握“报错排查四步法”
  - 学员理解“能跑通”是工程学习第一里程碑

## 二、课前准备

- 已安装基础命令行环境
- 进入项目目录：`/root/workspace/Agent/claw-code/rust`

## 三、授课流程（讲师时间轴）

## 0-15 分钟：生活类比与心态建设

### 讲师话术参考

“环境搭建像开餐馆前的后厨准备。  
你现在不是在做大菜，而是在确认煤气、刀具、冰箱全都能工作。  
这一步看起来慢，但决定后面是不是一路顺畅。”

## 15-40 分钟：演示标准流程

### 标准命令序列

```bash
cd rust
cargo build --workspace
cargo test --workspace
```

### 延伸验证

```bash
./target/debug/claw --output-format json prompt "ping"
```

### 讲师强调

- 先看“是否成功”，再看“是否优雅”
- 失败不是问题，不会复盘失败才是问题

## 40-60 分钟：报错排查四步法

### 方法

1. 记录现象（原始报错全文）  
2. 定位层级（工具链、依赖、配置、网络、权限）  
3. 执行修复（一次只改一个变量）  
4. 复验闭环（重新执行原命令）

### 课堂示例（讲师可口述）

- 工具链版本不兼容
- 缺少认证环境变量
- 命令路径错误

## 60-80 分钟：课堂实操

### 学员任务

1. 完成 build + test。  
2. 自己制造一个小错误（例如进入错误目录），再按四步法修复。  
3. 填写错误复盘模板。
